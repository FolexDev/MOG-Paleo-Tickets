Documentation

Guide complet du système de gestion de distribution des billets Paléo 2026 — Media One Group.

Connexion

L'accès à l'application est restreint aux comptes Google autorisés de Media One Group. Toute tentative de connexion avec un compte non autorisé affiche un message d'erreur.

1
Première connexion
Cliquer sur « Se connecter avec Google » et suivre les deux étapes : identification du compte, puis autorisation d'accès aux Google Sheets.
2
Reconnexion automatique
Lors des visites suivantes, la connexion est rétablie silencieusement sans intervention — le compte est mémorisé en local.
3
Déconnexion
Cliquer sur « Déconnexion » en haut à droite. La prochaine visite nécessitera une reconnexion manuelle.
Accès L'application est hébergée sur GitHub Pages et accessible depuis n'importe quel navigateur moderne. Aucune installation requise.

Interface générale

L'interface se compose de quatre zones principales :

  • En-tête — navigation vers les Google Sheets sources, bouton Actualiser, nom de l'utilisateur connecté, bouton Admin (réservé).
  • Barre d'onglets — sélection de l'antenne ou vue globale.
  • Barre de recherche — recherche en temps réel et badge d'envois possibles.
  • Tableau principal — liste des gagnants avec toutes les actions disponibles.

Le bouton Actualiser recharge les données depuis les trois Google Sheets et rafraîchit l'affichage. À utiliser après toute modification externe dans les Sheets.

Statuts

Chaque gagnant possède un statut qui reflète l'avancement de la distribution. Il est modifiable directement depuis le tableau via le menu déroulant de la colonne Statut.

StatutSignificationConditions
En attente Aucun contact établi Statut par défaut. Verrouillé si Nom, Tél et Email sont vides.
Contact fait Le mail d'information a été envoyé Passé automatiquement après un envoi via le bouton Envoyer.
A répondu Le gagnant a transmis ses informations Paléo Tickets Passé automatiquement après enregistrement des données Paléo Tickets.
Billets expédiés Les billets ont été transmis sur l'app Paléo Tickets À passer manuellement une fois les billets envoyés.
Transitions automatiques Deux transitions se font sans action manuelle : En attente → Contact fait lors d'un envoi mail via Make, et En attente / Contact fait → A répondu lors de l'enregistrement des données Paléo Tickets.

Actions sur une ligne

Modifier les informations gagnant

Cliquer sur l'icône crayon à côté du nom ou du jeu pour ouvrir la modale de modification. Permet de renseigner ou corriger : Nom & Prénom, Téléphone, Email, et Jeu/Origine. Ces données sont écrites directement dans le Google Sheet.

Copier l'email

L'icône de copie à droite de l'adresse email copie celle-ci dans le presse-papier. Un toast de confirmation apparaît en bas à droite.

Copier le modèle de mail

Le bouton Mail en fin de ligne copie le modèle de mail complet (format HTML + texte brut) dans le presse-papier. Le contenu est adapté automatiquement selon l'antenne (tutoiement pour One FM et Rouge, vouvoiement pour Radio Lac et LFM) et la date de la soirée.

Envoyer le mail via Make

Le bouton vert Envoyer est visible uniquement quand toutes ces conditions sont réunies :

  • Antenne ≠ Carac/MOC
  • Statut = En attente
  • Nom, Téléphone et Email tous renseignés
  • L'envoi est activé pour cette antenne (paramètre admin)

Un clic déclenche le webhook Make, qui envoie le mail depuis Gmail. Si l'envoi est confirmé par Make (réponse 200), le statut passe automatiquement à Contact fait dans le Sheet.

Données Paléo Tickets

La colonne Paléo Tickets affiche les données nécessaires au transfert des billets sur l'application Paléo Tickets : Nom, Prénom et Email du compte destinataire.

Cliquer sur l'icône de la colonne Paléo Tickets ouvre la modale de saisie. Selon la configuration (1 ou 2 billets), il est possible de :

  • Renseigner un seul compte (les deux billets sur le même compte)
  • Activer le split : renseigner deux comptes différents (1 billet par compte)
Transition automatique Après enregistrement des données Paléo Tickets, le statut passe automatiquement à A répondu — sauf s'il est déjà à Billets expédiés.

Envoi de mail

Contenu du mail

Le mail est généré automatiquement selon l'antenne et la date de la soirée. Il contient :

  • Salutation personnalisée avec le prénom du gagnant
  • Date de la soirée formatée
  • Liens vers l'application Paléo Tickets (Android & iOS)
  • Instructions pour transmettre les informations de compte
  • Signature spécifique à l'antenne

Recherche & filtres

Recherche

La barre de recherche en haut filtre en temps réel sur Nom, Prénom et Email dans tous les onglets. Les résultats sont surlignés en jaune. Les flèches de navigation permettent de passer d'un résultat au suivant.

Filtres par statut

Dans l'onglet Tous, des boutons filtres permettent d'afficher uniquement les gagnants d'un statut donné : En attente, Contact fait, A répondu, Billets expédiés.

Badge "envois possibles"

Le badge orange affiché à côté de la barre de recherche indique le nombre total de gagnants avec le statut En attente et toutes les données renseignées (prêts à être contactés). Un clic sur ce badge bascule vers l'onglet Tous filtré sur « A répondu ». Un second clic retire le filtre.

Modale admin

La modale Admin est accessible uniquement au compte administrateur via le bouton Admin dans l'en-tête.

Activation de l'envoi de mail

Deux toggles permettent d'activer ou désactiver l'envoi de mail via Make pour chaque groupe d'antennes :

  • One FM & Radio Lac
  • LFM & Rouge

L'état est stocké dans le Google Sheet (feuille ConfigHTML) et s'applique à tous les utilisateurs connectés. Désactiver masque le bouton Envoyer sur toutes les lignes concernées.

Journal des connexions

Les 50 dernières connexions sont affichées avec l'email et la date/heure. Ce journal est stocké localement sur l'appareil de l'administrateur (localStorage) — il n'est pas partagé.

Architecture

L'application est entièrement côté client — aucun serveur backend. Elle s'appuie sur :

  • Google Sheets API v4 — lecture et écriture des données via OAuth 2.0
  • Google Identity Services — authentification et autorisation
  • Make (webhook) — envoi des mails via Gmail
  • GitHub Pages — hébergement statique

Authentification

L'application utilise deux flux Google distincts :

  • Google Sign-In — identification de l'utilisateur (JWT)
  • OAuth 2.0 Token — autorisation d'accès en lecture/écriture aux Sheets

Après la première connexion, le compte est mémorisé en localStorage et le token OAuth est redemandé silencieusement à chaque visite.
